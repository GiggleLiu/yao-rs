## Expected vs Actual Outcome

All features achieved their expected outcomes. Numerical results matched quantum mechanics theory:
- Bell state: P(00)=0.5, P(11)=0.5, <ZZ>=1.0, <Z>=0.0, <P0>=0.5

## Issues Found

### Critical (blocks common user paths)

1. **Negative operator terms fail with `--op "-Z(0)"`** -- Clap interprets `-Z(0)` as a flag. Users must use `--op="-Z(0)"` or `-- -Z(0)`. Fix: add `allow_hyphen_values = true` to the `--op` clap attribute. (affects `run` and `expect` commands)

2. **Repo example circuit files incompatible with CLI** -- `visualization/example-basic.json` and `visualization/example-qft.json` use `"gates"` key without `"type"` discriminator. CLI requires `"elements"` with `"type": "gate"`. Error: `missing field 'elements'`.

### Important

3. **Circuit JSON format not documented in CLI help** -- No `--help` output shows the expected JSON schema. Users must find examples elsewhere or read source code.

4. **Bare `yao run circuit.json` dumps binary to stdout** -- Without `--shots`, `--op`, or `--output`, binary state data goes to stdout and garbles the terminal. Should warn on TTY.

5. **TN DTO indices are strings, not integers** -- Design spec shows `[[0, 1], [1, 2]]` but actual output is `[["0", "1"], ["1", "2"]]`. Could break downstream tools expecting integers.

### Minor

6. **Pu/Pd operator naming unclear** -- Help should clarify: Pu=|0><1| (raising/sigma+), Pd=|1><0| (lowering/sigma-)
7. **No CNOT gate alias** -- Users from Qiskit/Cirq expect "CNOT" or "CX" in circuit JSON
8. **Floating-point cosmetic display** -- Probabilities show `0.5000000000000001` instead of `0.5`
9. **`-0.0` in DM mode** -- Conjugation of zero produces `-0.0` in JSON imaginary parts
10. **Auto-JSON behavior undocumented** -- Docs should note that piped output auto-switches to JSON

## Suggestions

1. **Add `allow_hyphen_values = true`** to `--op` arg in `cli.rs` (fixes critical issue #1)
2. **Update `visualization/example-*.json`** to use `"elements"` format (fixes critical issue #2)
3. **Add circuit JSON schema** to `yao inspect --help` or a `yao example` command
4. **Warn when writing binary to TTY** in `simulate` and `run` (bare mode)
5. **Add a quickstart tutorial** with Bell state walkthrough to the mdBook docs
6. **Clarify operator names** in `expect --help`: add Pu/Pd/P0/P1 definitions
7. **Document auto-JSON** behavior in top-level help
